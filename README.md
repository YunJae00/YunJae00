## 시작하기

### 요구사항

프로젝트 실행을 위해 다음이 필요합니다:

- Node.js 18.0.0 이상
- npm 또는 Yarn
- MySQL 8.0 이상

### 설치 방법

1. **저장소 복제**
```bash
git clone https://github.com/username/project.git
cd project
```

2. **의존성 설치**
```bash
npm install
또는
yarn install
```

3. **환경 변수 설정**
`.env.local` 파일을 생성하고 다음 내용을 추가하세요:

```bash
NEXT_PUBLIC_API_URL=http://localhost:3000
DATABASE_URL=mysql://user:password@localhost:3306/dbname
NEXTAUTH_SECRET=your-secret-key
```

4. **개발 서버 실행**
```bash
npm run dev
또는
yarn dev
```

이제 `http://localhost:3000`으로 접속하여 프로젝트를 확인할 수 있습니다.

## 기여하기

프로젝트에 기여해주셔서 감사합니다. 🎉

### 기여 방법

1. 프로젝트를 포크합니다
2. 새로운 브랜치를 생성합니다 (`git checkout -b feature/amazing-feature`)
3. 변경사항을 커밋합니다 (`git commit -m 'feat: 새로운 기능 추가'`)
4. 브랜치에 푸시합니다 (`git push origin feature/amazing-feature`)
5. Pull Request를 생성합니다

### 커밋 컨벤션

| 타입 | 설명 | 예시 |
|------|------|------|
| feat | 새로운 기능 추가 | feat: 로그인 기능 구현 |
| fix | 버그 수정 | fix: 회원가입 유효성 검사 오류 수정 |
| docs | 문서 수정 | docs: README 업데이트 |
| style | 코드 포맷팅 | style: 들여쓰기 수정 |
| refactor | 코드 리팩토링 | refactor: 중복 코드 제거 |
| test | 테스트 코드 | test: 로그인 테스트 추가 |
| chore | 기타 변경사항 | chore: 패키지 버전 업데이트 |

## 배포 가이드

### 사전 준비사항

- AWS 계정
- Docker
- GitHub Actions 설정
- 도메인 (선택사항)

### 배포 스크립트

```bash
export AWS_REGION="ap-northeast-2"
export ECR_REPOSITORY="my-app"

도커 이미지 빌드
docker build -t $ECR_REPOSITORY .

AWS ECR 로그인
aws ecr get-login-password --region $AWS_REGION | docker login --username AWS --password-stdin $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com

이미지 태깅
docker tag $ECR_REPOSITORY:latest $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/$ECR_REPOSITORY:latest

이미지 푸시
docker push $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/$ECR_REPOSITORY:latest
```

### GitHub Actions 워크플로우

```yaml
name: Deploy to Production

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v1
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ap-northeast-2
      
      - name: Deploy to AWS
        run: |
          ./deploy.sh
        env:
          AWS_ACCOUNT_ID: ${{ secrets.AWS_ACCOUNT_ID }}
```

### 테스트 작성 가이드

```typescript
import { render, screen, fireEvent } from '@testing-library/react'
import { Button } from './Button'

describe('Button 컴포넌트', () => {
  it('클릭 이벤트가 정상적으로 동작해야 합니다', () => {
    // given
    const handleClick = jest.fn()
    render(<Button onClick={handleClick}>클릭</Button>)
    
    // when
    fireEvent.click(screen.getByText('클릭'))
    
    // then
    expect(handleClick).toHaveBeenCalledTimes(1)
  })

  it('disabled 상태에서는 클릭이 동작하지 않아야 합니다', () => {
    // given
    const handleClick = jest.fn()
    render(<Button disabled onClick={handleClick}>클릭</Button>)
    
    // when
    fireEvent.click(screen.getByText('클릭'))
    
    // then
    expect(handleClick).not.toHaveBeenCalled()
  })
})
```

### 보안 모범 사례

```typescript
// 잘못된 예시 - SQL Injection 취약점
const query = `SELECT * FROM users WHERE id = ${userId}`;

// 올바른 예시 - Prepared Statement 사용
const query = 'SELECT * FROM users WHERE id = ?';
await db.query(query, [userId]);

// 잘못된 예시 - XSS 취약점
element.innerHTML = userInput;

// 올바른 예시 - 안전한 텍스트 삽입
element.textContent = userInput;

// 잘못된 예시 - 안전하지 않은 암호화
const hash = crypto.createHash('md5').update(password).digest('hex');

// 올바른 예시 - 안전한 암호화
const hash = await bcrypt.hash(password, 10);
```

### 엔드포인트

#### 사용자 API

| 메소드 | 엔드포인트 | 설명 | 권한 |
|-------|------------|------|------|
| GET | /users | 사용자 목록 조회 | Admin |
| GET | /users/:id | 특정 사용자 조회 | Admin, User |
| POST | /users | 사용자 생성 | Admin |
| PUT | /users/:id | 사용자 정보 수정 | Admin, User |
| DELETE | /users/:id | 사용자 삭제 | Admin |
